## 🔍 误差累积规律

### 临界点发现

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  压缩层数          QA 准确率        状态
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   1 层              100%          ✅ 完美
  18 层 (50%)        100%          ✅ 完美
  36 层 (100%)         0%          ❌ 崩溃
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**关键洞察**:
- ✅ **安全区域**: 1-18 层（前 50%）
- ⚠️ **待验证**: 19-27 层（前 75%）
- ❌ **危险区域**: 28-36 层（全部）
- 🎯 **临界点**: 在 18-36 层之间

### 误差累积机制

**数学模型**:
```
第 i 层输出: H_i = f_i(H_{i-1})
压缩后: H_i' = f_i(H_{i-1}') ≈ H_i + ε_i
累积误差: Σε_i (i=1→N)
```

**实验验证**:
- N = 1: Σε ≈ 0 → 100% 准确率 ✅
- N = 18: Σε 小 → 100% 准确率 ✅
- N = 36: Σε 爆炸 → 0% 准确率 ❌

**结论**:
```
AM 不是架构问题（Qwen3 全 Attention）
AM 不是算法问题（单层完美）
AM 是层数问题（误差累积）
```

## 🔬 关键创新

### 1. Key-based Query Sampling

**问题**: MLX 不支持 hook 机制，无法实现 repeat-prefill

**解决**: 从 KV cache 的 keys 中采样作为 queries

```python
# 均匀采样 keys
indices = np.linspace(0, offset-1, num_queries, dtype=int).tolist()
indices_array = mx.array(indices, dtype=mx.int32)
sampled_queries = mx.take(full_cache[layer_idx].keys, indices_array, axis=2)
```

**优势**:
1. ✅ 避免 hook 机制问题
2. ✅ 真实反映 key 分布
3. ✅ 计算高效（直接 take）
4. ✅ 594 queries 足够覆盖

---

### 2. Bounded Least-Squares 求解器

**修复**:
```python
# 之前：伪逆（无界）
beta = np.linalg.pinv(R_S) @ target

# 现在：bounded LS
res = scipy.optimize.lsq_linear(
    R_S,
    target.flatten(),
    bounds=(beta_lower, beta_upper),  # [-3, 3]
    method='bvls'
)
beta = res.x
```

**结果**:
- β ∈ [-3, 3] ✅
- C2 拟合成功 ✅
- 质量完美保持 ✅
