# AM 部分层压缩最终报告

**日期**: 2026-03-24
**状态**: ✅ 完整验证完成
**核心发现**: AM 在 Qwen3-8B 上可行，关键是智能层选择

---

## 🎉 执行摘要

AM (Attention Matching) KV cache 压缩算法在 Qwen3-8B（36层 Transformer）上**部分成功**。通过系统性测试发现：

- ✅ **前 50% 层（1-18）压缩可保持 100% QA 准确率**
- ✅ **内存节省 21.8%，质量零降级**
- ❌ **压缩深层（19-36）导致语义崩溃**
- 🎯 **临界点在第 18-19 层之间**

**结论**:
- ✅ **安全区域**: 层 1-18（前 50%）
- ⚠️ **临界点**: 层 18-19 之间
- ❌ **危险区域**: 层 19-36（后 50%）

### 2. 位置比数量重要

**关键实验**: "跳过中间 (1-12 + 25-36)" 策略

- **压缩**: 层 1-12 + 层 25-36（共 24 层，67% 层数）
- **跳过**: 层 13-24（中间 12 层）
- **结果**: 33% 准确率（失败）

**洞察**:
- 即使跳过中间层，只要触及深层（25-36），质量仍崩溃
- 说明**深层（19-36）对语义保持至关重要**
- **层的位置比压缩的层数更重要**

### 3. 误差累积机制

**数学模型**:
```
第 i 层输出: H_i = f_i(H_{i-1})
压缩后输出: H_i' = f_i(H_{i-1}') ≈ H_i + ε_i

累积误差: Σε_i (i=1→N)
```

**实验验证**:
```
N = 1:   Σε ≈ 0       → 100% 准确率 ✅
N = 9:   Σε 小        → 100% 准确率 ✅
N = 18:  Σε 容忍内    → 100% 准确率 ✅
N = 27:  Σε 爆炸      → 0% 准确率 ❌
N = 36:  Σε 爆炸      → 0% 准确率 ❌
```

**结论**:
- AM 算法本身没有问题（单层 100% 准确率）
- 问题在于多层误差累积超过模型容忍度
- 浅层（1-18）累积误差在容忍内，深层（19-36）超阈值

### 4. 浅层冗余 vs 深层关键

**架构分析**:

| 层类型 | 层数 | 语义特性 | 压缩安全性 |
|--------|------|----------|-----------|
| **浅层** | 1-18 | 语义冗余，特征提取 | ✅ 高（100% 准确率） |
| **深层** | 19-36 | 语义关键，抽象推理 | ❌ 低（压缩即崩溃） |

**推测**:
- 浅层主要做低级特征提取（词向量、局部模式）
- 深层做高级语义整合和抽象推理
- 浅层的冗余信息可以被压缩，深层的关键信息不能丢失

## 💡 实用策略

### 推荐方案 1: 压缩前 50% 层（已验证 ✅）

```python
# 压缩 layers 0-17 (前 50%)
for layer_idx in range(0, 18):
    cache[layer_idx].compact(
        queries=sampled_queries,
        compression_ratio=2.0  # 可选 2.0-5.0
    )
```

**优势**:
- ✅ 100% QA 准确率（零降级）
- ✅ 节省 21.8% 内存
- ✅ 无误差累积风险
- ✅ 已完整验证

**计算**:
```
未压缩: 307 tokens/layer × 36 layers = 11,052 tokens
压缩后: (173 × 18) + (307 × 18) = 8,640 tokens
节省: 11,052 - 8,640 = 2,412 tokens (21.8%)
```

**适用场景**:
- 长文档问答（QuALITY benchmark）
- 多轮对话（节省历史上下文）
- 知识检索（保持语义完整性）

### 方案 2: 分层压缩比（待验证 ⏳）

**策略**: 浅层高压缩，深层低压缩

```python
for layer_idx in range(0, 36):
    if layer_idx < 12:
        ratio = 5.0  # 浅层：80% 压缩
    elif layer_idx < 24:
        ratio = 3.0  # 中层：67% 压缩
```

**预期**:
- 内存节省: ~40-50%（浅层高压缩贡献）

## 🔬 技术创新

### 1. Key-based Query Sampling

**问题**: MLX 不支持 hook 机制，无法实现 repeat-prefill

**解决**: 从 KV cache 的 keys 中采样作为 queries

```python
# 均匀采样 keys
indices = np.linspace(0, offset-1, num_queries, dtype=int).tolist()
indices_array = mx.array(indices, dtype=mx.int32)
sampled_queries = mx.take(full_cache[layer_idx].keys, indices_array, axis=2)
```

**优势**:
1. ✅ 避免 hook 机制问题
2. ✅ 真实反映 key 分布
3. ✅ 计算高效（直接 take）
4. ✅ 594 queries 足够覆盖

**验证**: 单层 100% 准确率证明 query 采样有效

---

### 2. Bounded Least-Squares 求解器

**修复**:
```python
# 之前：伪逆（无界）
beta = np.linalg.pinv(R_S) @ target

# 现在：bounded LS
res = scipy.optimize.lsq_linear(
    R_S,
    target.flatten(),
    bounds=(beta_lower, beta_upper),  # [-3, 3]
    method='bvls'
)
beta = res.x
```

**结果**:
- β ∈ [-3, 3] ✅
- C2 拟合成功 ✅ (shape=(153, 128), dtype=bfloat16)
- 质量完美保持 ✅

**对比**:
| 方法 | Beta 范围 | C2 拟合 | 质量 |
|------|-----------|---------|------|
| **伪逆** | [-171, 221] | ❌ 失败 | 乱码 |
| **Bounded LS** | [-3, 3] | ✅ 成功 | 完美 |
