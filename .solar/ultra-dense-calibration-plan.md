# 超密集校准文件生成计划

**优化目标**: 进一步提高校准精度，最小化 dynamic clipping 的影响

---

## 📊 校准文件密度对比

### v1 配置（原始）

```
文件数: 6 个
长度: L249, L466, L710, L944, L1403, L1863
步长: 200-450 tokens
文件大小: ~147-220 KB/个
总大小: ~1 MB
```

**问题**:
- 步长太大（200-450 tokens）
- 590 tokens → 匹配 L710（距离 120 tokens）
- Dynamic clipping 损失严重

---

### v2 配置（密集）

```
文件数: 10 个
长度: L300, L400, L500, L600, L700, L800, L1000, L1200, L1500, L2000
步长: 100-200 tokens
文件大小: ~147-220 KB/个
总大小: ~1.5-2 MB
```

**改进**:
- 步长减半（100-200 tokens）
- 590 tokens → 匹配 L600（距离 10 tokens）
- Dynamic clipping 损失减少

---

### v3 配置（超密集）✅ 当前

```
文件数: 25 个
长度: 
  - 300-800 (步长 50): L300, L350, L400, L450, L500, L550, L600, L650, L700, L750, L800
  - 800-1000 (步长 50): L850, L900, L950, L1000
  - 1000-2000 (步长 100): L1100, L1200, L1300, L1400, L1500, L1600, L1700, L1800, L1900, L2000

步长: 50-100 tokens
文件大小: ~147-220 KB/个
总大小: ~3.7-5.5 MB (仍然很小!)
```

**优势**:
- ✅ 步长极小（50-100 tokens）
- ✅ 任何长度都能精确匹配（误差 ≤ 50 tokens）
- ✅ Dynamic clipping 几乎无损失
- ✅ 总大小 < 6 MB（非常可接受）

---

## 🎯 精度提升分析

### 示例：590 tokens old_prefix
